## **Nano-Banana Pro — Compressed Prompting Guide for Image Description Generation**

**Model Capabilities Summary**  
Nano-Banana Pro is a _thinking_ image model optimized for **professional asset production**. It excels at:

- **Legible text rendering** and infographics
    
- **Character and identity consistency** (with multiple reference images)
    
- **High-resolution output (1K–4K)** and fine textures
    
- **Grounded generation using real-time Google Search**
    
- **Advanced editing** (inpainting, removal, restoration, colorization, localization)
    
- **Dimensional translation** (2D ↔ 3D)
    
- **Structured composition control** (sketches, wireframes, grids)
    
- **Story/sequence consistency** (storyboards, thumbnails, multi-image arcs)

---

## **Golden Rules (Apply These to All Image Descriptions)**

### **1. Edit, Don’t Re-roll**

Describe changes clearly instead of regenerating fully—models follow incremental edits extremely well.

### **2. Use Natural Language**

Write image descriptions like briefing a human artist. Use full sentences, clear intentions, and avoid “tag soup.”

### **3. Be Precise**

Include:

- **Subject** (who/what exactly)
    
- **Setting**
    
- **Lighting**
    
- **Mood**
    
- **Materiality/Textures**
    

### **4. Provide Context**

State _why_ or _for whom_ an image is made — it drives stylistic and compositional decisions.

---

## **Category-Specific Best Practices**

### **Text Rendering, Infographics, Visual Synthesis**

- Specify layout style: _editorial_, _technical diagram_, _whiteboard_, _retro_, etc.
    
- When including text, put it **in quotes**.
    
- Ask for “compressed data” or “visual summarization” for complex topics.
    

### **Character Consistency**

- Use explicit instructions like:  
    “Keep the facial features identical to Reference Image 1.”
    
- Define expression, pose, angle, and identity separately.
    

### **Search Grounding**

- For real-world data or events, ask for “visualization based on current trends / latest results.”
    

### **Advanced Editing**

- Use semantic instructions: “Remove the tourists,” “colorize,” “swap style,” “turn summer into winter.”
    

### **2D ↔ 3D**

- Provide floor plans, blueprints, sketches.
    
- Define the layout of multi-panel outputs.
    

### **High-Res & Textures**

- Explicitly request: “Render at 2K/4K resolution.”
    
- Mention micro-details (“seared crust,” “brushed steel,” “oak wood grain”).
    

### **Thinking & Reasoning**

- For problem-solving visuals (whiteboards, transformations), specify stepwise clarity.
    

### **Storyboards & Multi-Image Narratives**

- Set consistent identity, attire, world style.
    
- Define a 9-part or N-part sequence, generated image by image.
    

### **Structural Control**

- Provide sketches/wireframes to lock layout.
    
- For pixel art or sprite sheets: state grid size and constraints.

## **Universal Template for Image Descriptions (Optimized for This Model)**

When generating an image description, include the following:

**1. Goal/Context**  
Why the image exists (editorial, ad, story, diagram, etc.)

**2. Subject**  
Precise description of characters, objects, focal elements.

**3. Composition & Layout**  
Camera angle, framing, arrangement, grid or blueprint references.

**4. Setting & Environment**  
Location, era, objects, background, mood.

**5. Lighting & Atmosphere**  
Soft/hard, sunset/neon/overcast, warm/cold, cinematic, etc.

**6. Text (if any)**  
Write all text in _quotes_ and specify style (font, weight, placement).

**7. Style**  
Photorealistic, retro, editorial, anime, blueprint, whiteboard, pixel art, etc.

**8. Fidelity**  
Texture detail, resolution level, imperfections if needed.

**9. Consistency Notes**  
If recurring characters or styles: define identity locking.
