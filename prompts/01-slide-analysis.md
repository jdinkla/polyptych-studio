# Task: Comprehensive Slide Analysis

## Goal
Analyze a slide presentation generated by NotebookLM from a source essay, producing documentation sufficient to recreate similar slides from new essays.

## Inputs
- `source.md` - The original essay "The Cellophane World" (~2,500 words)
- `slides/slides1-01.png` through `slides/slides1-14.png` - AI-generated presentation slides

## Process

### Step 1: View All Slides
Read each slide image to understand:
- Text content (headlines, body, quotes)
- Visual composition and layout
- Imagery used and its relationship to the text
- Typography choices
- Color usage

### Step 2: Create Per-Slide Analysis Files
For each slide, create `analysis/slides/slide-NN.md` with these sections:

```markdown
# Slide [N]: [Title]

## Text Content
- Headline, subheadline, body text, quotes

## Visual Composition
- Layout type, grid structure, focal point, rule of thirds, white space

## Imagery
- Type (photo/illustration/diagram), subject, provenance (AI-generated/stock),
  mood, color temperature, metaphor mapping

## Typography
- Font families, weights, sizes, hierarchy, alignment

## Color Palette
- Background hex, text colors hex, image dominant colors hex

## Emotional Tone
- Mood, intensity (1-5), progression note

## Source Mapping
- Which paragraphs from source.md, extraction strategy
```

### Step 3: Create Design System Document
Synthesize patterns from all slides into `analysis/design-system.md`:
- Typography stack (font roles, sizes, weights)
- Color palette (hex codes with usage context)
- Layout patterns (with ASCII diagrams)
- Spacing and margin rules
- Image treatment guidelines

### Step 4: Create Content Mapping
Build `analysis/content-mapping.md` tracing source to slides:
- Paragraph-to-slide mapping table
- Extraction patterns (direct quote, paraphrase, synthesized header, etc.)
- Compression ratios
- Quote selection criteria
- Unused source material

### Step 5: Create Metaphor Lexicon
Catalog concept→visual mappings in `analysis/metaphor-lexicon.md`:
- Each metaphor with concept, visual, and slide number
- Categorize metaphors (transformation, container, path, body, statistical, object)
- Document emotional associations
- Note reusability for future projects

### Step 6: Create Narrative Arc Document
Analyze storytelling structure in `analysis/narrative-arc.md`:
- Three-act breakdown with emotional beats per slide
- Emotional intensity curve
- Pacing analysis (information density)
- Transition logic between slides
- Source reorganization (how essay order differs from slide order)
- Narrative techniques identified

## Key Findings

### Content Extraction Patterns
1. **Direct quote** - Verbatim from source
2. **Paraphrase with condensation** - Compressed while preserving meaning
3. **Synthesized header** - AI-invented titles capturing paragraph essence
4. **Conceptual expansion** - Brief phrase becomes structured visual
5. **Invented specifics** - AI creates concrete examples where source is abstract

### Compression Ratio
~2,500 source words → ~605 slide words = **4:1 compression**

### Narrative Reorganization
The AI moved paragraph 6 (redundancy fear) from mid-essay to slide 10 to serve as emotional nadir before resolution - demonstrating editorial judgment beyond simple extraction.

### Visual Innovation
Slide 13 ("Stumbling. Searching. Clumsy.") uses typography as imagery - wallpaper of rejected AI-words behind bold human-words. Form embodies content.

## Notes for Recreation

To generate similar slides from a new essay:

1. **Identify quotable passages** - Standalone clarity, metaphorical richness, rhythmic quality
2. **Extract/invent section headers** - Nominalize key concepts
3. **Map metaphors to visuals** - Use the metaphor lexicon as a starting reference
4. **Structure as three acts** - Problem → Confrontation → Resolution
5. **Vary information density** - Low-density slides at structural transitions
6. **Place emotional nadir before resolution** - Darkest moment makes hope brighter
7. **End with action** - Present tense, forward motion
